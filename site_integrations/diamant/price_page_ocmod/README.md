# Динамические цены на странице Diamant

Решение предназначено для OpenCart 3.0.3.2. Оно читает последнюю строку
`diamant_gold_prices` на сервере при открытии информационной страницы и заменяет
метки в HTML готовыми диапазонами цен.

JavaScript и публичный API не используются.

## Как работает

```text
Открытие страницы
  -> ControllerInformationInformation
  -> ModelCatalogInformation::getLatestGoldPrices()
  -> SELECT ... ORDER BY id DESC LIMIT 1
  -> замена {{gold_price_583}} и других меток
  -> готовый HTML пользователю
```

Запрос к таблице выполняется только для информационных страниц, в описании
которых присутствует строка `{{gold_price_`.

## Подготовленные файлы

```text
install.xml
diamant_dynamic_gold_prices.ocmod.zip
verify_latest_prices.sql
```

## 1. Сделать резервную копию

Перед установкой сохранить:

```text
catalog/controller/information/information.php
catalog/model/catalog/information.php
```

Также сохранить исходный HTML страницы из админки OpenCart.

OCMOD не изменяет эти файлы напрямую, но резервная копия упрощает диагностику.

## 2. Проверить таблицу

Открыть phpMyAdmin, выбрать базу Diamant и выполнить:

```text
verify_latest_prices.sql
```

Запрос должен вернуть одну последнюю строку с ценами.

## 3. Установить OCMOD

В админке OpenCart открыть:

```text
Расширения -> Установка расширений
```

Загрузить:

```text
diamant_dynamic_gold_prices.ocmod.zip
```

После загрузки открыть:

```text
Расширения -> Модификаторы
```

Нажать синюю кнопку обновления модификаторов.

## 4. Вставить метки в HTML страницы

Открыть страницу `Скупка золота в Ташкенте` в админке и перейти в режим
исходного HTML.

Заменить только содержимое блоков цен:

```html
<div class="prices-item-price">{{gold_price_583}} сум/гр</div>
<div class="prices-item-price">{{gold_price_585}} сум/гр</div>
<div class="prices-item-price">{{gold_price_750}} сум/гр</div>
<div class="prices-item-price">{{gold_price_850}} сум/гр</div>
<div class="prices-item-price">{{gold_price_875}} сум/гр</div>
<div class="prices-item-price">{{gold_price_916}} сум/гр</div>
<div class="prices-item-price">{{gold_price_crowns}} сум/гр</div>
<div class="prices-item-price">{{gold_price_999}} сум/гр</div>
```

`{{gold_price_crowns}}` временно использует диапазон 850 пробы.

Опционально можно вывести дату последней цены:

```html
<span>{{gold_prices_updated_at}}</span>
```

## 5. Очистить кэш

1. Обновить модификаторы OpenCart.
2. На главной странице админки открыть настройки разработчика через шестерёнку.
3. Очистить кэш темы и SASS.
4. Очистить кэш Journal3, если страница продолжает показывать старые данные.
5. Обновить публичную страницу через `Ctrl+F5`.

## 6. Проверить

Открыть:

```text
https://diamant.uz/skupka-zolota-v-tashkente
```

Сравнить значения с последней строкой `diamant_gold_prices`.

После изменения курса ботом и успешной синхронизации обновить страницу. Новые
цены должны появиться без ручного редактирования HTML.

## Fallback

Если таблицы нет или она пока пустая, OCMOD использует сохранённые резервные
диапазоны. Поэтому вместо цен не появятся пустые строки или необработанные метки.

## Откат

1. Вернуть в HTML обычные статические числа вместо меток.
2. В `Расширения -> Модификаторы` удалить `Diamant Dynamic Gold Prices`.
3. Нажать обновление модификаторов.
4. Очистить кэш OpenCart и Journal3.

Таблица истории и работа Telegram-бота при этом не изменяются.

## Если установщик OpenCart не принимает ZIP

Можно загрузить `install.xml` через стандартный OCMOD installer после упаковки
в ZIP либо установить XML вручную в таблицу модификаторов. Предпочтительный
вариант — стандартный установщик, чтобы модификацию можно было удалить из
админки без ручного редактирования файлов.
