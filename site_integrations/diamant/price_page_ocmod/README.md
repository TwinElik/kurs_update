# Динамические цены Diamant в Journal3

Проверено локально на OpenCart 3.0.3.2 и Journal3. Страница
`skupka-zolota-v-tashkente` выводится модулем Journal3 Blocks:

```text
module_id: 352
name: Landing Gold
```

OCMOD заменяет специальные метки в содержимом этого блока значениями последней
строки `diamant_gold_prices`. JavaScript и публичный API не используются.

## Схема

```text
Открытие страницы
  -> Journal3 Blocks / Landing Gold
  -> обнаружение {{gold_price_...}}
  -> SELECT * FROM diamant_gold_prices ORDER BY id DESC LIMIT 1
  -> серверная замена меток
  -> готовый HTML
```

Запрос выполняется только для Journal3-блоков, где найдена метка
`{{gold_price_`. Каталог, карточки товаров и остальные блоки не затрагиваются.

## Файлы

```text
diamant_dynamic_gold_prices.ocmod.zip
install.xml
verify_latest_prices.sql
```

Готовый HTML с метками находится локально:

```text
C:\Users\islom\Desktop\Text Document - dynamic prices.txt
```

Исходный `Text Document.txt` сохранён без изменений.

## 1. Резервная копия

В Journal3 открыть модуль `Landing Gold` и сохранить его текущий HTML отдельно.
Также сделать резервную копию БД или строки `ocoe_journal3_module.module_id=352`.

## 2. Проверить таблицу

В phpMyAdmin выбрать базу Diamant и выполнить `verify_latest_prices.sql`.
Запрос должен вернуть последнюю строку.

## 3. Установить OCMOD

1. Открыть `Расширения -> Установка расширений`.
2. Загрузить `diamant_dynamic_gold_prices.ocmod.zip`.
3. Открыть `Расширения -> Модификаторы`.
4. Нажать синюю кнопку обновления модификаторов.

## 4. Обновить Journal3 Landing Gold

Открыть в Journal3 модуль `Landing Gold`, перейти к HTML-содержимому блока и
вставить содержимое `Text Document - dynamic prices.txt`.

В нём используются метки:

```html
{{gold_price_583}}
{{gold_price_585}}
{{gold_price_750}}
{{gold_price_850}}
{{gold_price_875}}
{{gold_price_916}}
{{gold_price_crowns}}
{{gold_price_999}}
```

Единица `сум/гр` остаётся в HTML. Коронки временно используют диапазон 850
пробы. Дату последней строки можно вывести меткой `{{gold_prices_updated_at}}`.

## 5. Очистить кэш

1. Обновить модификаторы OpenCart.
2. Очистить кэш темы/SASS через шестерёнку разработчика.
3. Обязательно очистить кэш Journal3.
4. Обновить страницу через `Ctrl+F5`.

Без очистки Journal3 может продолжать отдавать ранее сохранённый HTML.

Рабочий `update-gold-price.php` после успешной записи автоматически удаляет
только файлы кэша `journal3.blocks.352.*`. Поэтому при следующих изменениях
курса ручная очистка не потребуется: следующий запрос страницы сформирует блок
из новой строки БД. Остальной кэш Journal3 не удаляется.

## 6. Проверить

Открыть:

```text
https://diamant.uz/skupka-zolota-v-tashkente
```

На странице должны быть ровно 8 карточек цен. Строк `{{gold_price_...}}` быть не
должно. Значения должны совпасть с последней строкой `diamant_gold_prices`.

## Защита от ошибок

- Если таблицы нет, используются резервные цены.
- Если таблица пустая, используются резервные цены.
- Запрос только читает данные и ничего не обновляет.
- Другие Journal3-блоки без меток не выполняют запрос.
- Исходный HTML можно вернуть за несколько минут.

## Откат

1. Вернуть сохранённый HTML модуля `Landing Gold`.
2. Удалить `Diamant Dynamic Gold Prices` из модификаторов.
3. Обновить модификаторы.
4. Очистить кэш Journal3 и OpenCart.

Таблица истории, бот и синхронизация сайтов при откате не изменяются.
