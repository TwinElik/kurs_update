# Алгоритм расчета цен скупки золота

Источник правил: Excel-файл. В код переносится не расположение ячеек, а бизнес-алгоритм.

## Главный курс

Пользователь вводит один главный курс. В Excel это значение находится в ячейке `B2`.

Один и тот же главный курс используется для всех брендов:

- `Goldexpert.uz`
- `Skupka-zolota.uz`
- `Diamant.uz`
- `Tillachi Bolla`

## Показываемые пробы

На картинках показываются:

```text
375, 583, 585, 750, 850, 875, 916, 999
```

Проба `900` может считаться отдельно, но в текущем дизайне не выводится.

## Стартовая цена

Для большинства проб:

```text
CEILING((sample / 583 / 10) * main_rate, 0.5) * 10000
```

Где:

- `sample` - проба;
- `main_rate` - главный курс;
- `CEILING(..., 0.5)` - округление вверх до шага `0.5`.

Для `585` отдельное правило:

```text
start_price = main_rate * 1000
```

То есть `585` стартует так же, как `583`.

## Максимальная цена "До"

Для `583` и `750`:

```python
if 1500000 - start_price < 200000:
    max_price = start_price + 200000
else:
    max_price = 1500000
```

Для остальных проб:

```python
max_price = roundup_to_10000(start_price) + brand_addition
```

`roundup_to_10000` округляет стартовую цену вверх до ближайших `10 000`.

## Важное правило генерации

Генератор картинок должен считать цены отдельно для каждого бренда:

```python
price_map = calculate_prices(main_rate, brand)
```

Нельзя один раз посчитать общий `price_map` и передать его всем четырем картинкам, потому что колонка `"До"` отличается по брендам.

Стартовые цены могут совпадать, но максимальные цены у брендов разные.
